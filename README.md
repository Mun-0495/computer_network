# AI를 활용한 채팅 프로그램 설계와 구현

이 프로젝트는 기존 `multi_server.cc`와 통신하는 채팅 클라이언트와, 그 기능을 확장한 서버를 함께 구현한 과제용 예제입니다. 핵심 목표는 단순히 메시지를 주고받는 수준을 넘어서, 실제 사용자가 더 편하게 사용할 수 있는 기능을 넣는 것이었습니다.

## 구현 요약

- 서버는 멀티스레드 구조를 유지합니다.
- 클라이언트는 송신 스레드와 수신 스레드를 분리해 동시에 입력과 출력을 처리합니다.
- TCP의 특성상 메시지가 한 번에 잘리지 않거나 합쳐질 수 있으므로, 서버는 줄 단위로 명령을 파싱합니다.
- 사용자 관점에서 바로 체감되는 기능을 우선 추가했습니다.

## 추가 기능

### 1. 닉네임 변경
- 명령어: `/nick 이름`
- 사용자가 대화의 정체성을 직접 만들 수 있도록 했습니다.
- 닉네임은 중복을 허용하지 않습니다.
- 같은 닉네임으로 다시 변경하려고 하면 중복으로 처리하지 않고 안내만 보여줍니다.

### 2. 접속 시 초기 닉네임 설정
- 클라이언트 실행 시 `NICKNAME` 인자를 주면 접속 직후 자동으로 `/nick` 명령을 보냅니다.
- 매번 서버에 들어온 뒤 바로 닉네임을 바꾸지 않아도 되어 편합니다.

### 3. 입장 및 퇴장 알림
- 누가 들어오고 나갔는지 바로 알 수 있어 대화 흐름을 놓치지 않습니다.
- 새로 들어온 사용자도 현재 상황을 파악하기 쉽습니다.

### 4. 귓속말
- 명령어: `/w 닉네임 내용`
- 전체 채팅이 부담스러운 상황에서도 특정 사용자에게만 메시지를 보낼 수 있습니다.

### 5. 액션 메시지
- 명령어: `/me 행동`
- 채팅창에서 현재 사용자의 동작을 자연스럽게 표현할 수 있습니다.
- 예: `/me 커피를 마시는 중`

### 6. 접속자 목록과 접속 시간
- 명령어: `/who`
- 현재 접속한 사용자와 접속 후 경과 시간을 보여줍니다.
- 대화 상대를 찾거나 방 분위기를 파악하는 데 도움이 됩니다.

### 7. 과거 메시지 검색
- 명령어: `/search 키워드`
- 저장된 채팅 로그에서 키워드가 들어간 메시지를 찾아볼 수 있습니다.

### 8. 도움말 제공
- 명령어: `/help`
- 처음 접속한 사용자도 바로 사용할 수 있도록 명령어 목록을 다시 보여줍니다.

### 9. 메시지 시간 정보 표시
- 모든 시스템 메시지와 대화 메시지에 시각이 표시됩니다.
- 대화 순서를 따라가기 쉽고, 기록성도 좋아집니다.

### 10. 채팅 로그 저장과 최근 대화 보기
- 서버는 공용 채팅과 시스템 이벤트를 로그 파일로 저장합니다.
- 새로 접속하면 최근 대화를 바로 보여줍니다.

### 11. 종료 처리
- 명령어: `/quit`
- 클라이언트에서 입력 종료 시 서버에 연결을 정리하고, `Ctrl+C`로도 종료할 수 있습니다.

## 사용자 관점에서의 설계 의도

이 프로그램은 “기능이 있다”보다 “사용자가 바로 이해하고 쓸 수 있다”를 더 중요하게 봤습니다.

- 처음 접속했을 때 바로 명령어 안내를 보여줍니다.
- 입장/퇴장, 닉네임 변경, 귓속말처럼 의미가 큰 이벤트는 시스템 메시지로 구분했습니다.
- `/me`와 `/search`를 추가해서 대화 표현과 로그 탐색이 가능해졌습니다.
- `/who`에 접속 시간을 넣어 단순 목록이 아니라 상황 파악용 정보로 만들었습니다.
- `/help`를 넣어 사용자가 기억하지 않아도 다시 확인할 수 있게 했습니다.
- 모든 메시지에 시간을 붙여 대화 맥락을 읽기 쉽게 했습니다.
- 닉네임 중복을 서버에서 걸러서 같은 이름 충돌을 막았습니다.
- 클라이언트 실행 시 닉네임을 인자로 넘기면 바로 접속 후 적용됩니다.

## 파일 구성

- [multi_server.cc](multi_server.cc): 멀티스레드 채팅 서버
- [multi_client.cc](multi_client.cc): 서버와 통신하는 클라이언트
- [subject.txt](subject.txt): 과제 설명

## 빌드 방법

아래 명령으로 두 프로그램을 컴파일합니다.

```bash
g++ -pthread -o multi_server multi_server.cc
g++ -pthread -o multi_client multi_client.cc
```

## 실행 방법

### 1. 서버 실행

```bash
./multi_server 12345
```

### 2. 클라이언트 실행

```bash
./multi_client 127.0.0.1 12345 tester
```

닉네임을 생략할 수도 있습니다.

```bash
./multi_client 127.0.0.1 12345
```

## 사용 가능한 명령어

- `/nick 이름` : 닉네임 변경
- `/w 닉네임 내용` : 귓속말
- `/me 행동` : 행동 메시지 표시
- `/search 키워드` : 저장된 채팅 로그 검색
- `/who` : 접속자 목록 확인
- `/help` : 명령어 도움말 다시 보기
- `/quit` : 종료

추가로, 클라이언트를 실행할 때 `[NICKNAME]` 인자를 붙이면 접속 직후 자동으로 닉네임이 설정됩니다.

입력할 때는 프롬프트 `>` 가 표시되어, 실제 채팅창처럼 바로 메시지를 입력하는 느낌으로 사용할 수 있습니다.

## 간단한 사용 예시

```text
사용자 A: /nick alice
사용자 A: 안녕하세요
사용자 B: /w alice 조용히 물어볼게 있어요
사용자 A: /who
사용자 A: /quit
```

## 제출용 메모

- 서버는 single thread로 바꾸지 않고, 클라이언트마다 스레드를 유지합니다.
- 클라이언트 기능이 늘어날수록 서버도 함께 업데이트해야 하도록 설계했습니다.
- 소스 파일 내부 주석에 구현 의도와 파싱 방식도 함께 적어 두었습니다.
